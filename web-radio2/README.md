# WEB-RADIO-2: Інструкція користувача та консольні команди (CLI)

Цей модуль призначений для керування інтернет-радіо та локальною медіатекою в системах OpenWrt/CyberWrt за допомогою швидких асинхронних AJAX-запитів. Сторінка інтерфейсу більше не перезавантажується повністю при виконанні дій.

Високопродуктивний та стабільний аудіоплеєр на мові C, розроблений спеціально для вбудованих систем (**embedded**) та роутерів під керуванням **OpenWrt** (сумісний з сучасними версіями ОС на базі ядер **5.15 – 6.x**). 

Проєкт використовує утиліти `curl` та `madplay` для декодування потокового аудіо та локальних файлів MP3.

---

## ✨ Особливості та покращення архітектури
* **Zero CPU/RAM Waste**: Повністю усунено важкі зовнішні виклики командної оболонки (`sed`, `awk`, `pidof`), які раніше створювали сотні фонових процесів та перевантажували слабкі MIPS/ARM процесори роутерів. 
* **Атомарний C-парсер**: Плейлісти `.m3u` обробляються порційно та порядково безпосередньо всередині C-коду через оперативну пам'ять (`/tmp`), зберігаючи ресурс Flash-пам'яті пристрою.
* **Анти-зависання мережі (Network Drop Protection)**: Інтегровано багаторівневий захист від обривів зв'язку. Якщо швидкість потоку падає нижче 1 КБ/с на час більше 10 секунд, плеєр автоматично перемикається далі.
* **Захист від «мертвих» станцій**: Якщо URL-адреса радіо містить помилку, повернула `404 Not Found` або сервер станції вимкнений, плеєр вийде по таймауту за 4 секунди і запустить наступний трек, не викликаючи зависання консолі.

## 🛠 Інтеграція в OpenWrt Buildroot

### 1. Розміщення файлів
У кореневому каталозі вашої системи збірки OpenWrt створить папку пакета:
```bash
mkdir -p package/utils/web-radio2/src
```
* Перемістіть усі файли вихідного коду (`main.c`, `play.c`, `opl.c`, `error_log.c`, `helpComand.c` та їхні `.h` заголовки) разом із внутрішнім `Makefile` у папку `package/utils/web-radio2/src/`.
* Головний Makefile пакета OpenWrt покладіть безпосередньо в `package/utils/web-radio2/Makefile`.

### 2. Вибір пакета у графічному меню
Запустіть конфігуратор:
```bash
make menuconfig
```
Перейдіть у розділ **`Utilities`**, знайдіть пакет **`web-radio2`** та натисніть `Пробіл`, щоб увімкнути його в образ прошивки (повинна з'явитися мітка `[*]`).

### 3. Компіляція
Для швидкої збірки лише цього пакета виконайте:
```bash
make package/utils/web-radio2/compile V=s
```
Готовий бінарний файл автоматично скопіюється в образ вашої прошивки за шляхом `/usr/bin/web-radio2`.

## 🎮 Керування та команди (Довідка)

Програма підтримує гнучке керування через аргументи командного рядка під час запуску в SSH-терміналі. Аргументи можна комбінувати та міняти місцями.

### Повний синтаксис аргументів:
* `-u [URL]` — Посилання на прямий аудіопотік (HTTP/HTTPS стрім радіостанції).
* `-n [Name]` — Кастомна назва радіостанції (використовується для виводу поточного треку у `/tmp/play_name`).
* `-p [№]` — Номер станції в плейлісті, з якої потрібно почати відтворення.
* `-f [Шлях]` — Шлях до конкретного локального файлу у форматі `.mp3`.
* `-l [Шлях]` — Шлях до папки з локальними аудіофайлами для їх послідовного відтворення.
* `-h` або `-?` — Виклик інформаційного меню довідки.

---

### 📝 Приклади реальних команд для терміналу:

#### 1. Циклічне відтворення всього загального плейліста
Запуск плеєра за замовчуванням. Програма почне грати файл `/etc/config/web-radio2` з першої станції. Після завершення останньої — автоматично повернеться на початок:
```bash
web-radio2
```

#### 2. Запуск плейліста з конкретної станції
Почати циклічне відтворення списку, пропустивши перші дві станції (старт з №3):
```bash
web-radio2 -p 3
```

#### 3. Відтворення однієї конкретної інтернет-станції (без плейліста)
```bash
web-radio2 -u http://stilnoe.fm
```

#### 4. Відтворення станції з присвоєнням кастомного імені
```bash
web-radio2 -u https://hitfm.ua -n "Hit FM Ukraine"
```

#### 5. Відтворення одного локального MP3-файлу з пам'яті роутера
```bash
web-radio2 -f /tmp/music/my_track.mp3
```

#### 6. Відтворення всієї папки з MP3-файлами
Плеєр послідовно відтворить усі аудіофайли, знайдені у вказаному каталозі:
```bash
web-radio2 -l /tmp/my_album
```

#### 7. Повне примусове зупинення плеєра
Оскільки плеєр працює у фоні, для повної зупинки музики та очищення мережевих потоків надішліть сигнал зупинки процесам:
```bash
killall -9 web-radio2 curl madplay
```

## 🗂 Структура системних файлів в OpenWrt
Під час роботи плеєр взаємодіє з такими шляхами в системі:
* `/etc/config/web-radio2` — Стандартний файл вашого плейліста у форматі `.m3u` (з тегами `#EXTINF`).
* `/tmp/play_name` — Тимчасовий файл в ОЗУ роутера, куди плеєр записує назву станції, яка грає в дану секунду (зручно для виводу назви на зовнішні екранчики або в Web-інтерфейс LuCI).
* `/tmp/Error_play_list` — Лог-файл помилок, куди записуються збої парсингу або критичні проблеми доступу до файлів плейліста.

## 📁 Структура файлів модуля

Всі файли повинні знаходитися в папці: `/www/cgi-bin/modules/web-radio2/`
1. `index.cgi` — Головний веб-інтерфейс (HTML + CSS + JS/AJAX).
2. `volume.cgi` — Фоновий обробник звукової карти (`amixer` для картки `0`, каналу `Line Out`).
3. `action.cgi` — Фоновий обробник команд плеєра (`madplay`, `curl`, `web-radio2`).

> **Важливо:** Після завантаження файлів на роутер обов'язково надайте їм права на виконання через SSH:
> ```bash
> chmod +x /www/cgi-bin/modules/web-radio2/*.cgi
> ```

## 🎛️ 1. Керування гучністю та звуком (`volume.cgi`)

Команди виконуються через передачу змінної оточення `QUERY_STRING`. Вони миттєво змінюють стан заліза через `amixer` 
та повертають відповідь у форматі: `ГУЧНІСТЬ|СТАТУС` (наприклад, `7|on`).

### Встановлення точного рівня звуку (параметр `set=`)
```bash
# Встановити гучність на рівень 7
QUERY_STRING="set=7" /www/cgi-bin/modules/web-radio2/volume.cgi

# Встановити гучність на рівень 15
QUERY_STRING="set=15" /www/cgi-bin/modules/web-radio2/volume.cgi

# Повністю прибрати звук (на 0)
QUERY_STRING="set=0" /www/cgi-bin/modules/web-radio2/volume.cgi
```

### Крокове регулювання (параметр `act=up` або `act=down`)
```bash
# Збільшити гучність на +1 крок:
QUERY_STRING="act=up" /www/cgi-bin/modules/web-radio2/volume.cgi

# Зменшити гучність на -1 крок:
QUERY_STRING="act=down" /www/cgi-bin/modules/web-radio2/volume.cgi
```

### Режим Mute / Unmute (параметр `act=mute`)
Працює як тригер-перемикач: перший виклик вимикає звук (`7|off`), другий — вмикає назад (`7|on`).
```bash
QUERY_STRING="act=mute" /www/cgi-bin/modules/web-radio2/volume.cgi
```

### Перевірка поточного стану без змін (параметр `act=get`)
```bash
QUERY_STRING="act=get" /www/cgi-bin/modules/web-radio2/volume.cgi
```

## 📻 2. Керування плеєром та плейлистом (`action.cgi`)

Оскільки цей файл приймає POST-запити від AJAX, для його виклику з консолі використовується утиліта `wget` (або `curl`), яка надсилає дані на локальну адресу роутера `http://127.0.0.1`.

### Перевірка поточного статусу (Що зараз грає?)
```bash
wget -qO- --post-data="cmd=get_status" http://127.0.0
```

### Керування живленням та потоком
```bash
# Запустити основний модуль радіо (START)
wget -qO- --post-data="zapusk=1" http://127.0.0

# Повністю зупинити програвання (STOP)
wget -qO- --post-data="ostanov=1" http://127.0.0

# Перемкнути на наступну радіостанцію (Next)
wget -qO- --post-data="next=1" http://127.0.0
```

### Прямий запуск (Потік / Файл / Папка)
```bash
# Запустити пряме інтернет-посилання URL
wget -qO- --post-data="URL=http://hostingradio.ru" http://127.0.0

# Запустити один конкретний локальний mp3 файл
wget -qO- --post-data="file=/home/Шансон/044.Очаровала.mp3" http://127.0.0

# Запустити програвання всієї папки з музикою
wget -qO- --post-data="folder=/home/Шансон" http://127.0.0
```

### Робота з каналами плейлиста (`/etc/config/web-radio2`)
```bash
# Увімкнути конкретну станцію зі списку за її номером (наприклад, Канал №3)
wget -qO- --post-data="nomerstroki=3&nameURL=Ретро-ФМ&URL2=http://retro.ru" http://127.0.0

# Додати нову станцію в кінець списку плейлиста
wget -qO- --post-data="add_nev_name=Зайцев-ФМ&add_nev_url=http://zaycev.ru" http://127.0.0

# Видалити канал зі списку за номером (наприклад, №2) з автоматичним перевпорядкуванням номерів
wget -qO- --post-data="delchan=2" http://127.0.0
```

### Керування автозапуском при старті роутера
```bash
# Увімкнути автозапуск модуля
wget -qO- --post-data="autostarton=1" http://127.0.0

# Вимкнути автозапуск модуля
wget -qO- --post-data="autostartoff=1" http://127.0.0
```

## 💡 Порада для швидкого доступу (Аліаси для консолі)

Щоб не вводити довгі команди вручну, додайте скорочення у файл `/etc/profile`:
```bash
alias vol7='QUERY_STRING="set=7" /www/cgi-bin/modules/web-radio2/volume.cgi'
alias volup='QUERY_STRING="act=up" /www/cgi-bin/modules/web-radio2/volume.cgi'
alias voldown='QUERY_STRING="act=down" /www/cgi-bin/modules/web-radio2/volume.cgi'
alias radiostop='wget -qO- --post-data="ostanov=1" http://127.0.0'
```
Після перезапуску терміналу або команди `source /etc/profile` керування зводиться до введення одного короткого слова (наприклад, `volup` або `radiostop`).
